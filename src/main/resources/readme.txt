Так как опыта работы над подобными проектами у меня нет и представление о некоторых вещах у меня всё ещё весьма смутное, то я постаралась по большей части не столько ответить на вопросы, сколько обозначить примерный план для предстоящего создания сетевого хранилища.

1. В данном проекте используется клиент-серверная архитектура, поэтому в первую очередь необходимо написать отдельно сервер и отдельно клиент, который и будет использовать пользователь. Для сетевого взаимодействия будет использоваться Netty

2. В первоначальных планах у меня создание консольного приложения. В дальнейшем я рассматриваю возможность изменения клиентской части и добавление примитивного графического интерфейса на JavaFX, состоящего из пары окон с минимумом функций.

3. Для подключения к серверу необходимо организовать авторизацию. Данные о пользователях (в частности логин, пароль и путь для хранения файлов на сервере) будут храниться в небольшой базе данных, к которой будет обращаться сервер. Так как база данных совсем небольшая, то я планирую использовать sqlite, чьих функций будет более чем достаточно.

4. Для выполнения каких-либо действий клиент будет посылать серверу различные служебные команды. К этим действиям относятся: просмотр файлов, размещенных на сервере, добавление нового файла, скачивание файла, удаление файла с сервера.

5. На первом этапе я планирую обойтись без каких-либо каталогов и вложенных папок. При первом подключении пользователя для него будет создаваться отдельная папка с уникальным названием, в которую он сможет сохранять свои файлы. Все файлы изначально будут храниться только в этой общей папке без какой-либо структуры.
В дальнейшем я пересмотрю этот момент и добавлю возможность пользователю создавать и удалять дополнительные папки.

Что касается непосредственной передачи файлов, я пока что могу только надеяться на то, что в ближайшее время смогу узнать каким образом это реализовать.
